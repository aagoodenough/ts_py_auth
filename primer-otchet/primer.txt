1.2. Описание текущего состояния студии
Музыкальная студия обслуживает учеников всех возрастных категорий: дети, подростки, взрослые. Штат включает директора и нескольких преподавателей, каждый из которых специализируется на определённых направлениях. Студия располагает четырьмя учебными кабинетами, оснащёнными музыкальными инструментами, ноутбуками и вспомогательным оборудованием.
На момент проведения анализа учёт в студии ведётся преимущественно вручную:
–	посещаемость фиксируется в бумажном журнале;
–	абонементы учитываются в Google Таблицах без автоматической обработки;
–	контакты учеников и родителей хранятся в личных телефонах директора и педагогов;
–	расписание составляется каждым педагогом самостоятельно в произвольном формате;
–	заявки от новых клиентов не фиксируются  всё происходит устно по телефону.

1.3.4. Управление абонементом
–	Абонемент покупается по факту обращения вручную директором.
–	Активируется со дня покупки сроком на 30 дней.
–	При пропуске занятия оно списывается со статусом «Пропущено».
–	При остатке 1–2 занятий система автоматически уведомляет ученика о продлении.

1.3.5. Управление расписанием
–	Каждый педагог самостоятельно выстраивает своё расписание.
–	Ученики идут «подряд» без промежутков; стандартный блок  60 мин.
–	Изменения в расписании происходят часто; система должна поддерживать перенос.
–	Директор контролирует распределение кабинетов.
–	Система визуально отображает свободные слоты по кабинетам и педагогам.

1.4. Автоматизация бизнес-процессов
По результатам интервью определены процессы, подлежащие автоматизации:
–	Полуавтоматическое составление расписания с визуализацией по кабинетам (аналог Google Календаря).
–	Автоматические напоминания ученику (за 24 ч и за 1 ч) и педагогу (за 24 ч).
–	Напоминание педагогу о неотмеченном занятии через 2 ч после его окончания.
–	Автоматическое уведомление об окончании абонемента при остатке ≤ 2 занятий.
–	Ежедневный cron-скрипт перевода абонементов в статус «Истёк» при наступлении даты окончания.
–	Автоматический расчёт заработной платы педагогов на основе проведённых занятий.

3.4. Обоснование выбора инструментов и языка разработки
Выбор технологического стека обусловлен следующими факторами.
–	TypeScript как основной язык разработки обеспечивает статическую типизацию сквозь весь стек (фронтенд, бэкенд, ORM), что снижает количество ошибок на этапе разработки.
–	Next.js 14 позволяет объединить фронтенд и бэкенд в одном приложении, что существенно упрощает архитектуру для небольшой команды.
–	PostgreSQL выбрана за надёжность, поддержку транзакций и открытый исходный код. Реляционная модель данных хорошо подходит для предметной области с чёткими связями (ученик ↔ абонемент ↔ занятие).
–	Prisma ORM автоматически генерирует TypeScript-типы из схемы БД, что исключает рассинхронизацию типов между бэкендом и БД.
–	tRPC обеспечивает сквозную типизацию между клиентом и сервером без необходимости описывать DTO вручную.
–	FullCalendar.js выбран за встроенную поддержку режима ресурсов (resource timeline), позволяющего отобразить все кабинеты в виде параллельных колонок с визуализацией занятых слотов.
–	pg-boss позволяет реализовать очередь фоновых задач непосредственно в PostgreSQL, не добавляя Redis как отдельный сервис.

4.5. Схема Google OAuth 2.0
–	Пользователь нажимает «Войти через Google».
–	Редирект на страницу Google для выбора аккаунта.
–	Пользователь даёт разрешение.
–	Google возвращает code на /api/auth/callback/google.
–	Сервер обменивает code на токен, получает профиль пользователя.
–	Если email уже есть в БД  привязывает google_id, иначе создаёт новую запись.
–	Устанавливается httpOnly cookie сроком 7 дней.
–	Редирект на /dashboard.

6.2.1 Листинг-Рега
'use client';

import Link from 'next/link';
import { useRouter } from 'next/navigation';
import { useState } from 'react';
import styles from '../page.module.scss';

export default function RegisterPage() {
  const router = useRouter();
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState('');

  async function handleSubmit(e) {
    e.preventDefault();
    setError('');
    setLoading(true);

    try {
      const res = await fetch('/api/auth/register', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ name, email, password }),
      });

      const data = await res.json();

      if (!res.ok) {
        setError(data.error || 'Ошибка регистрации');
        return;
      }

      router.push('/');
    } catch {
      setError('Не удалось подключиться к серверу');
    } finally {
      setLoading(false);
    }
  }

  return (
    <div className={styles.page}>
      <div className={styles.card}>
        <div className={styles.header}>
          <h1>Регистрация</h1>
          <p>Создайте аккаунт, чтобы продолжить</p>
        </div>

        <form className={styles.form} onSubmit={handleSubmit}>
          <div className={styles.field}>
            <label htmlFor='name'>Имя</label>
            <input
              id='name'
              type='text'
              placeholder='Иван Иванов'
              value={name}
              onChange={(e) => setName(e.target.value)}
              required
              autoComplete='name'
            />
          </div>

          <div className={styles.field}>
            <label htmlFor='email'>Email</label>
            <input
              id='email'
              type='email'
              placeholder='you@example.com'
              value={email}
              onChange={(e) => setEmail(e.target.value)}
              required
              autoComplete='email'
            />
          </div>

          <div className={styles.field}>
            <label htmlFor='password'>Пароль</label>
            <input
              id='password'
              type='password'
              placeholder='Не менее 8 символов'
              value={password}
              onChange={(e) => setPassword(e.target.value)}
              required
              minLength={8}
              autoComplete='new-password'
            />
          </div>

          {error && <p className={styles.error}>{error}</p>}

          <button className={styles.submit} type='submit' disabled={loading}>
            {loading ? 'Регистрация...' : 'Создать аккаунт'}
          </button>
        </form>

        <p>
          Или <a href='/api/auth/google'>зарегистрироваться через Google</a>
        </p>
        <p className={styles.footer}>
          Уже есть аккаунт? <Link href='/'>Войти</Link>
        </p>
      </div>
    </div>
  );
}

'use client';

  const router = useRouter();
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState('');

  async function handleSubmit(e) {
    e.preventDefault();
    setError('');
    setLoading(true);
